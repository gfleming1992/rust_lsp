# 6.2 FileRevision

The `FileRevision` element tracks changes to the IPC-2581 file. The revision identifier does not necessarily track the revision of the product but establishes the sequence and the software tools used to make the changes. The attributes of a `FileRevision` element are defined as follows:

| Attribute / Element Name | Attribute / Element Type   | Description                                                                                                                     | Occurrence |
|--------------------------|----------------------------|---------------------------------------------------------------------------------------------------------------------------------|------------|
| FileRevision             | FileRevisionType           | Tracks the changes that have been made to an IPC-2581 file.                                                                      | 1-1        |
| fileRevisionId           | string                     | An identifier for the revision. May be supplied by a revision control system such as RCS, CVS, or SCCS.                          | 1-1        |
| comment                  | string                     | A short description of the revision, such as a change statement entered by RCS or SCCS.                                           | 1-1        |
| label                    | string                     | A label that can be applied to a branch head; used to associate a file revision of special significance.                        | 0-1        |
| SoftwarePackage          | SoftwarePackageType        | A nested element, the software package that wrote this revision of the file.                                                     | 1-1        |

**Example:**

```xml
<FileRevision fileRevisionId="Example1" comment="Primitive layout positioning" label="BetaRelease">
    <SoftwarePackage name="Manual Interpretation" vendor="IPC" revision="none"/>
</FileRevision>
```
