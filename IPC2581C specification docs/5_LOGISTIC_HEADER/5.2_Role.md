# 5.2 Role

A `Role` element declares a type of activity within an enterprise. The attribute values of the `Role` are based on the requirements of the activities performed by the role.

| Attribute / Element Name | Attribute / Element Type | Description                                                                                                                                                                                                                                            | Occurrence |
|--------------------------|--------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------|
| Role                     | RoleType                 | Defines the type of activity within the enterprise.                                                                                                                                                                                                    | 1-n        |
| id                       | string                   | A string that must be consistent throughout the IPC-2581 file; a globally unique name that identifies the specific role responsibility associated with the general role descriptions.                                                                     | 1-1        |
| roleFunction             | roleFunctionType         | Defines a globally unique identification of the role within an enterprise. The name is a restricted string that must be unique within the global namespace. Standard IPC role types include:
- `SENDER`
- `OWNER`
- `RECEIVER`
- `DESIGNER`
- `ENGINEER`
- `BUYER`
- `CUSTOMERSERVICE`
- `DELIVERTO`
- `BILLTO`
- `OTHER`                                                                                                                                     | 1-1        |
| description              | string                   | Further defines a role within an enterprise to highlight specific capabilities of the `roleFunction`.                                                                                                                                                    | 0-1        |
| publicKey                | base64Binary            | The public encryption key for the role, base64 encoded. If present, data can be encrypted so that only someone with the roleâ€™s private key can access it.                                                                                              | 0-1        |
| authority                | string                   | The access level associated with this role as defined by the system referenced by an external configuration control entry point.                                                                                                                        | 0-1        |

**Example:**
```xml
<LogisticHeader>
  <Role id="CircuitDesign" roleFunction="ENGINEER" description="In charge of Impedance Control"/>
  <Role id="LayoutPerson" roleFunction="ENGINEER"/>
  <Role id="PurchasingNo1" roleFunction="BUYER" description="To be informed of cost modification"/>
  <Role id="LayoutEngineer" roleFunction="DESIGNER"/>
</LogisticHeader>
```
