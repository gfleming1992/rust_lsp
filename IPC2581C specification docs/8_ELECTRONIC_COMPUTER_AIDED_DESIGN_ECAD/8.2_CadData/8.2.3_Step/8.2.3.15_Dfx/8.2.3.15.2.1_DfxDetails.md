### 8.2.3.15.2.1 DfxDetails

The DfxDetails element is used to specify any feature, layer, spec, image, or any other attribute found in 
FeatureDescription needed to identify a DFx issue. 
 
#### Attributes

| Attribute /  Element Name | Attribute /  Element Type | Description | Occurrence |
|---|---|---|---|
| DfxDetails | DfxDetailsType | A nested element containg any details pertaining to a measurement, query, or response | 0-1 |
| FeatureDescription | FeatureDescriptionType | An optional element describing the feature, location, image, or reference of interest for Dfx | 0-n |
| Marker | MarkerType  | An element that contains data to define a graphical marking to identify the point(s) of interest. Uses of  Marker include a boundary around the point(s) of interest or an identifting flag, such as a bow tie | 0-n |
| EmbeddedRef | EmbeddedRefType | The EmbeddedRef element holds the file is base64 encoded. The data is encoded using base64. (see IETF RFC 1421 for the base64 algorithm and http://www.w3.org/TR/xmlschema 2/#base64Binary) | 0-n |
| ExternalRef | anyURI | An attribute that defines a reference to an external or source of data such as a URL, specific file or document location. See the examples below. | 0-n |

```xml
<ExternalRef>file:///C:/testmethod.avi</ExternalRef> 
<ExternalRef>http://ipc2581.com</ExternalRef> 
```
