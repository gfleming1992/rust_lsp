# 8.1.1.6 General

The `General` specification defines general product characteristics (electrical, thermal, material) and instructions or standards influencing the product.

| Attribute / Element Name | Attribute / Element Type | Description                                                                                                      | Occurrence |
|--------------------------|--------------------------|------------------------------------------------------------------------------------------------------------------|------------|
| General                  | GeneralType              | Identifies a general specification instance.                                                                      | 0-n        |
| type                     | generalListType          | ELECTRICAL | THERMAL | MATERIAL | INSTRUCTION | STANDARD | CONFIGURATION | OTHER. If OTHER, `comment` is required. | 1-1        |
| comment                  | string                   | Description supplementing or differing from the enumerations.                                                     | 0-1        |
| Property                 | PropertyType             | Numerical/text attributes related to the general requirement.                                                     | 0-n        |
| ColorGroup               | ABSTRACT                 | Optional substitution to assign color via ColorTerm/ColorRef/RGB.                                                | 0-1        |

Examples:
```xml
<Spec name="Standards">
  <General type="STANDARD">
    <Property text="Telcordia Standard XYZ" refUnit="CLASS" refValue="3"/>
  </General>
  <General type="STANDARD">
    <Property text="IPC-4101" refUnit="SECTION" refText="8.2.1.13"/>
  </General>
</Spec>

<Spec name="Soldermask">
  <General type="MATERIAL">
    <Property text="SMOBC-LPI"/>
    <ColorTerm name="GREEN"/>
  </General>
</Spec>

<Spec name="Build Notes">
  <General type="INSTRUCTION">
    <Property text="Line neckdowns are intentional, build as is." refUnit="ITEM" refValue="1"/>
    <Property text="100 ohm differential pair traces that violate the specified separation are intentional, build as is." refUnit="ITEM" refValue="2"/>
  </General>
</Spec>
```
